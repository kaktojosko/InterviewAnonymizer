# Interview Anonymizer 🎭

**Текущая версия: v3.0.0 Stable.**

Interview Anonymizer — это высокопроизводительный локальный сервис для полной автоматической анонимизации видео-интервью. Инструмент использует передовые алгоритмы машинного обучения для надежного скрытия лиц (сверхбыстрый детектор YuNet) и изменения голоса (Pitch Shifting), обеспечивая 100% приватность.

Специально оптимизирован для работы с длинными видео (часовые интервью) на обычных процессорах (CPU) благодаря агрессивной параллелизации и умным алгоритмам пропуска кадров.

## ✨ Ключевые особенности

* **Экстремально быстрый ML**: Используется легковесная нейросеть **YuNet (OpenCV)** (всего 1.7 МБ). Детекция происходит на сжатой до 480p копии кадра, что дает прирост скорости.
* **Математическая интерполяция**: Нейросеть сканирует лишь каждый 28-й кадр. Позиции блюра в промежуточных кадрах плавно и точно высчитываются математически. Встроена защита от "перепрыгивания" блюра между лицами.
* **Zero-Copy Архитектура**: Видео читается напрямую через `ffmpeg readinto` с кольцевым буфером, что полностью исключает выделение памяти (Zero-Allocation) и копирование в горячем цикле.
* **Умная работа с потоками**: Прямая нарезка видео на чанки (без демуксинга) и параллельное искажение голоса в фоне. Адаптивный чанкинг автоматически делит видео на оптимальные куски. 
* **Выбор количества обработчиков**: В интерфейсе можно выбрать количество воркеров для конкретного запуска: `1`, `2`, `4`, `6`, `12`, `16` или свое значение. Максимум воркеров не означает максимум производительности: оптимальное значение зависит от CPU, охлаждения, памяти и конкретного видео.
* **Анонимизация голоса**: Быстрое искажение тембра (pitch shifting) с помощью нативных аудио-фильтров FFmpeg (`asetrate`, `atempo`) без полной загрузки в оперативную память, работает параллельно с обработкой видео.
* **Высококонтрастный Claymorphism UI**: Современный, отзывчивый и потрясающе красивый веб-интерфейс с поддержкой Светлой и Темной темы (сохраняется в localStorage), а также живым логом процесса обработки без задержек.

## 🛠 Технологический стек

* **Backend**: Python 3.10, FastAPI, Uvicorn
* **Computer Vision**: OpenCV (cv2), YuNet (ONNX)
* **Audio Processing**: FFmpeg Native Filters (ffmpeg-python)
* **Video/Audio Manipulation**: FFmpeg, ffmpeg-python
* **Frontend**: HTML5, Vanilla JavaScript, CSS3 (CSS Variables, Claymorphism)
* **DevOps**: Docker, Docker Compose

## 🚀 Установка и запуск (через Docker)

Проект полностью контейнеризирован. Вам не нужно устанавливать Python или FFmpeg локально, достаточно установленного [Docker](https://www.docker.com/) и [Docker Compose](https://docs.docker.com/compose/).

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/your-username/interview-anonymizer.git
   cd interview-anonymizer
   ```

2. Соберите и запустите контейнер в фоновом режиме:
   ```bash
   docker-compose up --build -d
   ```

3. Откройте интерфейс в браузере:
   ```text
   http://localhost:8000
   ```

## ⚠️ Хранение файлов и освобождение места

Приложение работает локально и хранит рабочие файлы на вашем компьютере:

* `uploads/` — загруженные исходные видео. Обычно удаляются после завершения обработки, но могут остаться после аварийной остановки.
* `output/` — готовые анонимизированные видео. Они остаются на диске, чтобы их можно было скачать повторно.
* `tmp/` — временные чанки и промежуточные файлы обработки. Папка очищается при старте контейнера и после задач, но после сбоя Docker или выключения компьютера там могут остаться данные.

Длинные видео могут быстро занимать десятки гигабайт. Периодически проверяйте размер папок `uploads/`, `output/` и `tmp/`, особенно после тестовых прогонов.

### Очистка файлов приложения

Перед удалением убедитесь, что нужные готовые видео уже скачаны: очистка `output/` удалит результаты обработки.

PowerShell:

```powershell
docker compose down
Remove-Item -Recurse -Force .\uploads\* -ErrorAction SilentlyContinue
Remove-Item -Recurse -Force .\output\* -ErrorAction SilentlyContinue
Remove-Item -Recurse -Force .\tmp\* -ErrorAction SilentlyContinue
docker compose up -d
```

Bash:

```bash
docker compose down
rm -rf uploads/* output/* tmp/*
docker compose up -d
```

## 🧹 Очистка Docker-кэша

Docker хранит build cache, старые слои образов и остановленные контейнеры. После частых пересборок или больших видео кэш может занимать много места. Текущий `Dockerfile` копирует в образ только код приложения, а `.dockerignore` исключает `uploads/`, `output/` и `tmp/`, но build cache Docker всё равно нужно иногда чистить.

Посмотреть, сколько места занимает Docker:

```bash
docker system df
```

Безопасная очистка build cache:

```bash
docker builder prune
```

Более агрессивная очистка build cache:

```bash
docker builder prune --all --force
```

Если сборка падает с ошибкой вида `parent snapshot ... does not exist` или `failed to prepare extraction snapshot`, очистите BuildKit-кэш и пересоберите образ:

```bash
docker compose down
docker builder prune --all --force
docker compose build --no-cache web
docker compose up -d web
```

Осторожно с глобальной очисткой:

```bash
docker system prune --all
```

Эта команда удаляет неиспользуемые образы, остановленные контейнеры и сети Docker во всей системе, а не только в этом проекте. Не используйте `--volumes`, если не уверены, что вам не нужны Docker volumes других проектов.

## ⚙️ Конфигурация системы

Все ключевые настройки вынесены в файл `src/config.py`. Вы можете изменять их под свои нужды:

```python
# Настройки отступа блюра от краев лица
FACE_PADDING_PERCENT_X = 0.35
FACE_PADDING_PERCENT_Y = 0.35

# Порог уверенности нейросети YuNet (0.3 - оптимально для мелких лиц)
YUNET_MIN_CONFIDENCE = 0.3

# Сканировать каждый N-ный кадр (увеличьте для скорости, уменьшите для плавности)
DETECT_EVERY_N_FRAMES = 28

# Ограничение FPS обработки. 30 FPS снижает нагрузку на CPU для VFR/60 FPS видео.
MAX_PROCESSING_FPS = 30.0

# Параметры H.264 кодирования обработанных чанков
OUTPUT_VIDEO_PRESET = os.environ.get("OUTPUT_VIDEO_PRESET", "ultrafast")
OUTPUT_VIDEO_CRF = os.environ.get("OUTPUT_VIDEO_CRF", "26")

# Размер куска видео для параллельной обработки (в секундах)
CHUNK_DURATION_SECONDS = 60 

# Количество параллельных воркеров по умолчанию.
# Значение из UI ограничивается логическими CPU минус один.
CPU_COUNT = os.cpu_count() or multiprocessing.cpu_count() or 1
MAX_ALLOWED_WORKERS = max(1, CPU_COUNT - 1)
MAX_WORKERS = max(1, min(MAX_ALLOWED_WORKERS, int(os.environ.get("VIDEO_MAX_WORKERS", CPU_COUNT - 3))))

# Смещение тональности голоса (в полутонах. Отрицательные значения делают голос ниже)
PITCH_SHIFT_SEMITONES = -2.0
```

### Подбор количества воркеров

Начинайте с `4-6` воркеров на обычных 6-ядерных CPU и сравнивайте общее время обработки одного и того же видео. Важен не пик загрузки CPU, а строка `Total time` в логах. Если первые чанки идут значительно дольше последних, значит система перегружена и количество воркеров стоит уменьшить.

Приложение не позволит выбрать больше `(количество логических CPU - 1)`, чтобы оставить системе хотя бы один поток для ОС, Docker и браузера.

## 📁 Структура проекта

```text
├── src/
│   ├── models/            # Легковесные ONNX модели (YuNet)
│   ├── audio_processor.py # Обработка и pitch-shifting звука
│   ├── config.py          # Основной конфигурационный файл
│   ├── ffmpeg_utils.py    # Вспомогательные функции для демуксинга/муксинга и склейки
│   ├── io_manager.py      # Управление папками (jobs, uploads, output)
│   ├── orchestrator.py    # Оркестратор параллельной обработки (нарезка, пулинг, сборка)
│   └── video_processor.py # Основное ядро: YuNet детекция, интерполяция, блюр
├── static/
│   ├── index.html         # Главная страница (Claymorphism UI)
│   ├── script.js          # Логика UI (Темы, загрузка файлов, поллинг логов)
│   └── style.css          # Стили тем и анимации
├── app.py                 # FastAPI роуты и сервер
├── docker-compose.yml     # Настройки контейнеризации
├── Dockerfile             # Сборка образа (установка системного ffmpeg и python пакетов)
└── requirements.txt       # Python зависимости
```
